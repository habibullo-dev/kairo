## Context

Phases 1–9 built a safe, evaluated, multi-surface agent with real data flowing in (connectors, digest, vault). But everything lives in one undifferentiated global pool: UI chats vanish (never persisted), memory is one bucket, there is no way to run a team of models against a problem, and cost is a status-bar number that evaporates. **Phase 9B (calendar/drive writes) is explicitly deferred behind this phase** — the user wants the project-based workstation and orchestration/cost layer first.

Phase 10 delivers: **Projects** as the first-class unit of work (chats, memory, tasks, KB sources, repos, settings scoped to them); **Modes** (Plan/Approval/Auto) enforced by backend policy, never bypassing the PermissionGate/taint substrate; a **model/provider registry** (roles → model routes, Anthropic + OpenAI-text-only, fail-closed on missing keys); a **persistent per-model-call cost ledger** with budgets and a Costs screen; and an **AI Orchestration Studio** — council → synthesis → execution → review → verdict workflows built directly on Phase 6's `SubAgentService` (no second agent framework).

**User decisions locked in (2026-07-07):** two sub-milestones with a **mandatory Checkpoint C between 10A and 10B**; OpenAI = **text-only analysis roles** (executors stay Anthropic; full tool-use shim deferred); ledger = **LLM completions only, labeled** (embeddings/voice excluded by documented design, never silently); run records **keep verbatim prompt/result in DB but never render them on cost/studio surfaces** (titles sanitized, sweep extended).

**Approval amendments (2026-07-07, binding):**
- **A1 — KB/Vault project scoping is IN scope, not deferred.** Project A must not retrieve Project B's KB content: `query_knowledge_base` respects the active scope (project P → `project_id = P OR project_id IS NULL`; global → `project_id IS NULL` only); ContextSelector validates every selected KB/source id as project-owned or global; pre-Phase-10 sources stay global; adversarial no-leak tests required. If full chunk-level migration is hard, ship the smallest correct source-level filter that blocks cross-project retrieval — KB is not left as a global leak.
- **A2 — Raw prompt/result storage gets an explicit privacy + retention rule.** `model_calls` stays metadata-only; Studio/Costs/orchestration read models never expose verbatim prompts/results by default; any raw transcript storage is explicit, local-only, retention-capped by config, redaction/sweep-tested, reachable only via a debug-only path, and clearly distinguished from summaries/manifests; raw provider error bodies and tokens/secrets/keys are never stored; tests prove Studio/Costs/Daily/Projects APIs return no raw prompt bodies, email bodies, secrets, or connector payloads.
- **A3 — Voice project binding is explicit.** The voice surface displays the active project when a voice turn starts; if no project can be established, voice runs in global scope (or requires screen selection before any project-scoped memory/task write); voice still prepares, screen commits; a test proves voice reflection/memory writes cannot land in the wrong project after a project switch.
- **A4 — PLAN_SAFE is visible and pinned**: the exact list appears in the Checkpoint C evidence report, and adding any new tool forces a deliberate classification.
- **A5 — Ledger degradation is visible**: a ledger write failure never breaks a turn, but surfaces a clear `ledger_degraded` notice/status (NoticeBoard + status strip + Hub) so cost tracking never silently disappears.
- **A6 — Honest live-connector status**: Phase 9 live verification is not claimed complete in docs/UI until the user runs the live checklist (see pre-flight note).

**Pre-mortem corrections baked into this plan (architecture-changing — treat as decided):**
1. **Auto mode lives at the approver seam, not a gate wrapper.** Taint-persistability (`Decision.persistable=False`) is computed in the loop *after* `gate.check()`; a gate-level Auto would auto-approve tainted egress and reopen the Phase 9 exfil pipe. Plan-mode denial = gate wrapper; Auto-approval = approver that sees the post-taint decision.
2. **Plan mode is allowlist-based** (`PLAN_SAFE` frozenset, pinned) — "deny side-effecting" would fail open for future unclassified tools.
3. **Memory scoping is a write-path change too**: `project_id` threaded through `remember`/`add`/`Provenance`; reflection attributes to the session's project; a project session never writes NULL. **Dedup uses exact-scope predicates** (project writes only compare against that project; global against global) — the recall union (`project_id=? OR IS NULL`) would let cross-scope supersede corrupt data. Global-scope recall = `project_id IS NULL` only.
4. **Council/review roles get a hard read-only floor**: `READ_ONLY_SPAWNABLE = {read_file, list_dir, glob_search, query_knowledge_base}` — no shell (never read-only), no write, **no web egress** (a prompt-injected council context must not be able to exfiltrate itself).
5. **Ledger = `LedgeredClient` wrapper** (same wrapper pattern as UnattendedGate, at the client layer), applied once in the `ClientFactory` so all completion paths + the OpenAI adapter are tapped uniformly; zero changes to `AnthropicClient` or the ~450 FakeClient fixtures. Pricing keyed by `(provider, model)`; prefix-matching only within Anthropic (dated snapshots); `price_for`-style fail-closed on all ledger/display paths (unknown → NULL + warning, **never silent $0**).
6. **Backend ownership validation on every new mutation**: any context/chat/memory/KB id in a request must resolve to a row owned by the request's project (or global where allowed) — UI filtering is not a control.
7. **`cost_context` contextvar set INSIDE each child coroutine** (gather copies context at task creation) with `token = set(...)` / `reset(token)` discipline for sequential stages.
8. **Execution stage holds the shared turn lock** (it is a writer); council/synthesis/review/verdict run off-lock (the `_fire_digest` pattern). `orchestration_runs` gets a startup orphan sweep (`running`→`aborted`) mirroring `agent_runs`.
9. **Budgets reserve worst-case before parallel fan-out** (width × per-child ceiling) — a between-stages check alone lets a council overspend N×.
10. **A session belongs to exactly one project for its lifetime**; switching projects starts a new session (otherwise reflection/promote attribute mixed content to the wrong project). Mode is snapshotted per turn (a flip to Auto never applies to an in-flight turn; a stricter flip applies from the next iteration).
11. **Memory import ignores inbound front-matter `project_id`/`source`** — forces `project_id = active project`, `source='import'` (untrusted); export paths jailed via `safe_wiki_path`; export/import are human rituals, never reachable by any agent/orchestration role.
12. **OpenAI adapter maps usage fields explicitly** (`prompt_tokens→input_tokens` etc.) and validates response shape (fail loudly) — otherwise every OpenAI call silently costs $0 and an empty response reads as success.

## 1. Resolved design decisions

### D1 — Projects & migration v7

`projects` table: `id, name, slug UNIQUE, description, status CHECK(active|paused|archived), color, icon, repos_json, settings_json, created_at, updated_at, archived_at`. Nullable `project_id INTEGER REFERENCES projects(id)` added to `sessions, memories, tasks, kb_sources, digests, agent_runs`, plus `sessions.pinned INTEGER NOT NULL DEFAULT 0` and `agent_runs` columns `orchestration_run_id, role, stage`. All plain `ADD COLUMN` — no FK-off rebuild (nothing widens a CHECK). Create `orchestration_runs` and `model_calls` **before** the `agent_runs` ALTERs in the script (FK ordering). Indexes: `sessions(project_id, updated_at)`, partial `memories(project_id) WHERE status='live'`, `model_calls(project_id, ts)`, `model_calls(orchestration_run_id)`. Never-DELETE: archive is a status flip. `settings_json` holds per-project overrides only (model routes, budgets, roster tweaks) — **never keys**. NULL `project_id` = global scope; existing data stays global (no adoption migration).

`ProjectContext(project_id, name, repos, system_extra)` is *surface state* (UiSession / REPL hold the active project), injected into `AgentLoop` as a callable provider feeding `build_system(extra=)` / `_system_with_extras`. The system extra is framed as data ("Active project: …"), not instructions. New sessions inherit the active project at creation and keep it for life; switching projects in the UI starts a new session. **Voice binding is explicit (A3)**: the voice surface announces/displays the active project when a voice turn starts; if no active project can be established, the voice session runs in **global scope** — project-scoped memory/task writes from voice require the project to have been selected on screen first (voice prepares, screen commits — unchanged). A voice session, like any session, is pinned to its project for life; a project switch on screen starts a new voice session binding. Sub-agent children inherit `project_id` for audit only (they keep `memory=None` — pinned).

### D2 — Project chats (the UI persistence fix)

`SessionStore` gains `list_sessions(kind, project_id, pinned, limit, offset)`, `search_sessions(query, project_id)` (SQL LIKE over title + message content — no embeddings), `set_pinned`, `set_project`, `create_session(..., project_id=None)`. `UiSession` lazily creates a session row on first turn and calls `save_messages` after each completed turn inside the held turn-lock window (delete+reinsert already re-arms reflection correctly); `resume(session_id)` mirrors REPL `--resume` mechanics (messages + compaction). REPL `--resume` (`latest_session_id`, interactive-only) must remain unhijacked — pinned. Reflection now picks up UI sessions naturally (kind=interactive). Routes: `GET /api/sessions`, `GET /api/sessions/{id}`; mutations `POST /api/sessions/{id}/pin`, `POST /api/sessions/{id}/resume`.

### D3 — Project-scoped memory + KB retrieval (recall ∪ global; dedup exact-scope; import untrusted)

- **Recall predicates**: active project P → `(project_id = P OR project_id IS NULL)`; global scope → `project_id IS NULL` only. Filter applied in SQL before the numpy matmul (smaller candidate matrix, same mechanics).
- **Write path**: `MemoryService.remember(..., project_id)` → `MemoryStore.add`; `Provenance` carries it; reflection reads the session row's project. A project chat's memories are always project-tagged; only truly global sessions write NULL.
- **Dedup**: the nearest-neighbor search inside `remember` uses **exact scope match only** — a project write can never supersede a global memory or another project's, and vice versa (pinned adversarially).
- **UI**: `GET /api/memory?project_id=` — "What Kairo knows about this project"; editable via existing forget + the new human-authority remember route.
- **KB retrieval scoping (A1)**: `kb_sources.project_id` (v7) is enforced at retrieval, not just recorded. New ingests are tagged with the active project (NULL when global); pre-Phase-10 sources stay global. `KnowledgeService.search` (behind `query_knowledge_base`) filters candidate chunks **by their source's scope** with the same predicates as memory: project P → `project_id = P OR IS NULL`; global → `IS NULL` only — a source-level SQL filter before scoring (the smallest correct filter; no chunk-table migration needed since chunks join to sources). The wiki/write paths keep their existing rules; ContextSelector (10B) validates KB/source ids as project-owned-or-global. Adversarial tests: text ingested only into Project B is never retrievable from Project A or global scope (string-level assertion on tool output).
- **Export/import** (`projects/export.py`, human ritual only — CLI `jarvis project export|import <slug>`): one `.md` per memory under `exports/<slug>/memories/`, front matter via `build_front_matter` (stable id, foreign Obsidian keys preserved), filenames via the `safe_wiki_path` jail. Import: match by id → update, else create; **ignore** inbound `project_id`/`source`/`confidence` — force active project, `source='import'`; dedup within target scope only. Never exposed to any tool/agent/orchestration role (pinned).

### D4 — Modes (two seams, not one)

Mode enum: `plan | approval | auto`. Debug stays a UI presentation flag (not a gate mode); voice sessions are pinned to `approval`; the BackgroundRunner keeps `UnattendedGate` untouched — Auto can never leak into unattended runs.

| Inner result | plan | approval | auto |
|---|---|---|---|
| ALLOW, tool ∈ PLAN_SAFE | ALLOW | ALLOW | ALLOW |
| ALLOW, tool ∉ PLAN_SAFE | **DENY** ("plan mode") | ALLOW | ALLOW |
| ASK reaching the approver, decision `persistable` ∧ tool ∈ `modes.auto_allow_tools` ∧ tool ∉ {run_shell, write_file} ∪ NEVER_PERSIST ∪ SUBAGENT_HARD_DENY | (never reached — plan denies at the gate) | human ASK | **auto-ALLOW** (audited `mode_auto_approved`, visible in Trace — never hidden) |
| ASK otherwise (incl. tainted-egress `persistable=False`) | DENY | human ASK | human ASK |
| DENY | DENY | DENY | DENY |

- **PlanModeGate** wraps the interactive `PermissionGate` (the UnattendedGate wrapper precedent); `PLAN_SAFE` is an enumerated frozenset pin — adding a tool forces a deliberate classification (mutation-route-pin discipline).
- **AutoApprover** wraps the surface's approver and sees the **post-taint** decision — tainted egress and any non-persistable ASK always reach the human. `auto_allow_tools` defaults `[]` (opt-in); `run_shell`/`write_file` are excluded even if configured (config cannot widen — pinned).
- Mode snapshotted at `run_turn` start for its permissive effects; stricter switches apply from the next iteration. Every decision/audit line carries `mode`; `POST /api/mode`; WS `mode_changed`; status-bar chip. New audit events (`mode_changed`, `mode_auto_approved`) join `AUDIT_EVENTS`.

### D5 — Model/provider registry + OpenAI text-only adapter

Roles + routes are **code constants** (`models/roles.py`): `ROLES` = planner, coder, reviewer, security, ux, qa, researcher, docs, judge, utility, voice_stt, voice_tts; `ModelRoute(provider, model, effort, text_only)`. Defaults: planner/judge → `claude-fable-5`; coder → `claude-opus-4-8`; utility → `claude-sonnet-5` (matching today's ModelsConfig). Resolution: `DEFAULT_ROUTES` ← `settings.yaml model_routes:` ← `projects.settings_json["model_routes"]` ← per-run override; the resolved snapshot (no keys) goes into `orchestration_runs.config_json`. Registry validation **rejects** a `text_only` route on any write-capable role.

`OpenAIChatClient` implements `LLMClient.create` text-only: raises typed `UnsupportedToolUseError` if `tools` is non-empty; returns a single text block with `stop_reason="end_turn"`; **explicit** usage mapping (`prompt_tokens→input_tokens`, `completion_tokens→output_tokens`); non-empty-content validation (fail loudly); latency measured. openai SDK is already a dependency (voice extra + dev group); lazy import.

`ClientFactory`: caches instances keyed `(provider, effort, thinking)`; missing provider key → `ConfigError` naming the env var (fail-closed, reuses `Secrets` machinery); per-role Anthropic effort = distinct client instances (the `_utility_client thinking=False` precedent). The factory is the single place the `LedgeredClient` wrap is applied. Hub/registry UI: presence booleans only (the existing sweep auto-covers new GETs).

### D6 — Cost ledger + versioned pricing (fail-closed, completions-only)

`config/pricing.yaml`: `schema_version`, `effective` date, `cache_multipliers`, `models` keyed by provider+model with `{input, output}` per MTok. Loader in `cost.py`; the code `PRICES` table remains the fallback for known Anthropic models (malformed/missing yaml → fallback + warning; never invented prices). Lookup keyed by provider; prefix matching **only** within anthropic (dated snapshots); other providers exact-match; unknown → `None`. `cost_of`'s legacy unknown→0.0 stays for its existing callers; the ledger and every display path use the `price_for` fail-closed semantics.

`model_calls` (the ledger; **metadata only — never prompts/bodies/secrets**): `ts, trace_id, session_id, project_id, orchestration_run_id, agent_role, purpose (turn|subagent|compaction|reflection|memory_dedup|digest|orchestration), provider, model, effort, input/output/cache_write/cache_read tokens, tool_call_count, latency_ms, cost_usd (NULL = unpriced, NEVER silent 0.0), pricing_version, created_at`.

`LedgeredClient(inner, ledger, provider, effort)` implements `LLMClient`; records one row after each `create` from `cost_context` (ContextVar; set inside child coroutines with set/reset tokens; nested utility calls override `purpose`). Ledger write failures log and never break the model call — **and are never silent (A5)**: a failure flips a `ledger_degraded` status surfaced on the NoticeBoard, the status strip, and Hub ("cost tracking degraded since <ts> — N unrecorded calls"), cleared on the next successful write; a `pricing_unknown` warning event fires for every unpriced row. Scope: **LLM completions only** — Voyage embeddings and voice STT/TTS are documented exclusions (voice already has its own egress-byte accounting); the Costs screen labels the total "LLM completion cost" and shows "unpriced calls: N" separately. The existing `model_call` JSONL log line stays (debug trail); the table is the queryable ledger.

`BudgetService` (`observability/budget.py`): `BudgetsConfig` — `soft_warn_usd_per_run`, `hard_stop_usd_per_run`, `project_monthly_usd`, `per_role_max_usd`, `max_rounds`, `max_parallel`, `cheap_fallback_routes`, `confirm_above_usd`, `hourly_rate_usd`, per-workflow `baseline_minutes`; project overrides in `settings_json`. Rollups = SQL SUMs over `model_calls` (day/week/month, local tz). Unpriced rows count as *unknown* — `budgets.treat_unpriced_as_blocking: true` (default) refuses orchestration with unpriced role models unless an explicit, audited per-run override.

### D7 — Orchestration engine on `SubAgentService.spawn` (10B)

`spawn()` gains backward-compatible keywords: `client=None, model=None, role=None, stage=None, orchestration_run_id=None, project_id=None, fresh_trace=False`. The `spawn_agent` **tool** passes none of them — model routing stays config-only, never model-controllable (tool schema pinned). `agent_runs` writes the new columns; `_child_config` handles the model override; per-role clients arrive pre-wrapped from the ClientFactory. A fresh trace per stage avoids the per-turn spawn-cap counter; parallel council spawns each get their own `_IN_SUBAGENT` context (bounded by `Semaphore(max_parallel)`); depth-1 stands.

Roster/workflows are frozen dataclass **code constants** (`orchestration/roles.py`, `workflows.py`); project roster customization = `settings_json` overrides *validated against the same invariants* at load (invalid → refuse the run, never silently widen). Invariants pinned by tests: ≤1 `write_capable` role per template; council/review stages ⊆ `READ_ONLY_SPAWNABLE` (no shell/write/**web**); every role's tools ⊆ `SPAWNABLE`. Ten templates: plan_feature, implement, review_diff, security_review, ux_critique, research, release_notes, debug_eval, refactor_proposal, council_review. (The `research` workflow's researcher role may carry web tools — but then its context bundle excludes cross-source private content; the same agent never holds both.)

Engine flow (`engine.py`): `start()` resolves roster/routes (fail-closed on unpriced/unkeyed), estimates cost, inserts the run row, launches an `asyncio.Task` **off the turn lock**. **A Council**: parallel spawns, identical framed `ContextBundle` per role. **B Synthesis**: direct head-route client call, forced-tool JSON schema (judge.py pattern), council reports as framed untrusted inputs. **C Execution**: acquires the shared turn lock; ONE write-capable child through the existing `SubAgentGate` + UI approver (Gate modal, no bypass — plan-mode surfaces refuse to start execution-class workflows); lock released after; a WS event explains why chat is briefly queued. **D Review**: parallel read-only spawns over the produced diff/report. **E Verdict**: forced schema accept/reject/revise; revise loops C–D up to `max_rounds`. Engine state transitions read **run records, never report text** (forged child output can't alter status). Cancellation → children cancelled, run `cancelled`; crash → the startup sweep marks `aborted`. `ContextSelector` output: bundle framed untrusted per agent; manifest (paths/ids/hashes/token counts, **no bodies**) → `context_manifest_json`. Backend validates every selected id belongs to the run's project (403 otherwise).

`orchestration_runs`: `project_id, workflow, title (sanitized — never raw user/email text), config_json, context_manifest_json, status CHECK(running|ok|rejected|revise|error|cancelled|aborted|budget_stopped), stage, verdict, synthesis_summary (short), estimated/actual/budget cost, session_id, trace_id, started_at, finished_at, created_at`.

**Raw transcript policy (A2)**: `model_calls` is metadata-only, always. `orchestration_runs` rows hold summaries + manifests only. Verbatim material exists in exactly two explicit, local-only places — `agent_runs.prompt/result_text` (existing Phase 6 behavior) and child session transcripts — and both get: (a) a retention cap `orchestration.raw_retention_days` (config; a startup sweep clears prompt/result_text on orchestration-linked rows older than the cap — never-DELETE holds for the audit columns, only the raw text fields are nulled); (b) redaction at write time — provider error bodies, tokens/secrets/keys never stored (reuse the friendly-error discipline); (c) a debug-only access path (existing Agents/Trace debug surfaces), never the Studio/Costs/Daily/Projects read models or WS. Tests prove those four API families return no raw prompt bodies, email bodies, secrets, or connector payloads (canary-seeded, sweep-style).

**Budget enforcement points**: pre-start (monthly breach → refuse; `est > confirm_above_usd` → two-step confirm on the run mutation); **worst-case reservation before each parallel stage** (width × per-child token ceiling; refuse if over remaining); between every spawn/stage (soft → WS warn + optional cheap-fallback for remaining *read-only* roles only — never the executor, pinned; hard → `budget_stopped`); per-role cap checked before that role's spawn. Estimate math: per read-only agent `est_in = manifest_tokens + ~1200 overhead`, `est_out ≈ 3000`; synthesis/verdict in = Σ council out + overhead; execution ×3 iteration multiplier; displayed as a range `[est, min(3×est, hard_cap)]`, cache ignored (conservative); refuse if any role model unpriced (unless the audited override).
