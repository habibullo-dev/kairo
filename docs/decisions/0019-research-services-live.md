# ADR-0019: Research services live (Phase 13)

*Status: accepted (Phase 13, 2026-07-09). The five service rows built in 10B (ADR-0015) are flipped
on for real use behind the existing fail-closed machinery; the S7 context-reuse layer (ADR-0018)
is turned on behind a default-OFF flag. Live-verified under Habib's Task-10 approval.*

## Context

Phase 10B built the `SERVICE_CATALOG` + a fail-closed `ServiceRegistry` + a `ServiceTool` base that
DERIVES gate/taint/framing from the spec, and shipped only three local, keyless adapters. Phase 13
turns on the hosted research capability those rows describe — without weakening any data-flow
boundary — and enables the S7 prompt/context-cache layer for live cost/latency savings.

## Decision

**Five adapters, all `ServiceTool` subclasses (policy derived, never hand-set):**
- `firecrawl_scrape` — URL → clean markdown (hosted; `public_only`, egress, ASK, execution-only,
  `untrusted_external_content`). `firecrawl_crawl` deferred (async/multi-page — a larger runaway
  surface; the plan marks it optional).
- `exa_search` — query → ranked snippets (`x-api-key`; hard `max_results ≤ 10`).
- `searxng_search` — meta-search via a LOCAL loopback SearXNG (no key; still egress — it proxies
  out; a non-loopback base URL is refused).
- `generate_image` — prompt → PNG saved under the managed `data/artifacts` root and registered
  as an artifact (`untrusted_model_generated`, never executed/committed). Requires the artifact
  store to be composed (else unavailable).
- **`jina_read` DEFERRED** — it cleared no material value bar over the now-shipped `firecrawl_scrape`
  + the free `web_fetch`; stays `priority: later`. `JINA_API_KEY` is not read.

**Shared substrate:** `HttpServiceTool` (injectable transport; friendly errors that never leak the
provider body; a key read consistent with the registry via `.env`→`Secrets`→`os.environ`);
per-unit + per-run/per-day **cost caps** (`ServiceBudget`, checked BEFORE each metered call);
per-project **narrow-only** service selection (`POST /api/projects/{id}/services`, subset-enforced;
run-time refusal for a narrowed-out tool); a **read-only settings** policy surface.

**S7 enable-step:** `context_reuse.enabled` (default OFF; flag-off byte-identity keeps replay
deterministic). On ⇒ the live Anthropic client marks a `cache_control` breakpoint on the STABLE,
NON-SENSITIVE system prefix only (memory recall / project / summary sit after it, never cached);
OpenAI gets a `prompt_cache_key`. Compat providers + Gemini + **Z.ai** get nothing.

## The walls (unchanged; live-verified — see docs/verification-13.md)

- **B1** — the five are `public_only`; the engine refuses any private/project/repo provenance for
  them. Private content cannot reach a research service; the request carries only its narrow args.
- **B2** — every fetched/generated byte is framed untrusted; a planted injection survives inside
  the frame as inert data, and a capable model treats it as data (does not act on it).
- **Egress + taint** — all five are `egress=True` (derived); a private read this turn demotes an
  egress ALLOW to a non-persistable ASK; outside PLAN_SAFE; never Auto-approved; unattended
  HARD-DENY.
- **Fail-closed availability** — unknown/missing-key/unpriced/deferred/narrowed-out ⇒ the tool
  never registers. Real per-unit pricing rows or the service stays blocked.
- **Caps are hard** — a metered call that would breach the run/day cap is refused before it is
  sent, with a reason (never truncated silently).
- **Image assets are data** — under the managed root, `untrusted_model_generated`, never executed.
- **Context reuse cannot widen data-flow** — stable non-sensitive prefix only; flag default OFF.

## Z.ai (optional worker, disabled)

Z.ai is cataloged as an OPTIONAL worker provider only. It is disabled by default (no default route
targets it; its key was absent this session ⇒ fail-closed), context reuse is OFF for it, and it is
never a final-authority route (planner/judge/utility are anthropic-only). No context reuse, authority,
high-risk, or connector-write role for Z.ai until its API behavior, pricing, and reliability are
verified.

## Consequences

Kairo can now research the live web and generate design assets under the same derived, fail-closed
enforcement as everything else, with hard cost caps and per-project narrowing, and can cache stable
prompt prefixes for cost/latency — all opt-in, all reversible by a config edit. ADR numbering: 0019
(Phase 13). Next reserved: 0020 (Phase 14).
